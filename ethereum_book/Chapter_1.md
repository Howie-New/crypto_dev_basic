## 什么是以太坊


[上一章：术语](Glossary.md)

以太坊常被描述为"世界计算机"。这究竟意味着什么？让我们先从计算机科学的视角切入，再以更务实的分析来解读以太坊的能力与特性，并将其与比特币及其他去中心化信息交换平台（更准确地说，即*区块链*）进行比较。

从计算机科学的视角看，以太坊是一台确定性但实际上无边界的状态机（State Machine），由两部分组成：一个全局可访问的单例状态（Singleton State），以及一台对该状态施加变更的虚拟机（Virtual Machine）。

从更务实的视角看，以太坊是一套开源的、全球去中心化的计算基础设施，用于执行被称为*智能合约*（Smart Contract）的程序。它借助区块链来同步和存储系统状态的变更，并使用名为*以太币*（Ether）的加密货币来计量和约束执行资源的成本。

以太坊平台使开发者能够构建具有内置经济功能的强大去中心化应用。它提供高可用性、可审计性、透明性和中立性，同时减少或消除审查，并降低特定的交易对手风险。

<a id="bitcoin_comparison"></a>

### 以太坊与比特币的比较


许多人是带着加密货币（尤其是比特币）的经验来接触以太坊的。以太坊与其他开放区块链有许多共同要素：连接参与者的点对点（P2P）网络、用于状态更新同步的拜占庭容错共识算法、数字签名和哈希等密码学原语，以及数字货币（以太币）。

然而，无论从目的还是构造来看，以太坊都与之前的开放区块链（包括比特币）有着显著差异。

以太坊的首要目的并非充当数字货币支付网络。尽管以太币对以太坊的运行既不可或缺又至关重要，但它本质上是一种*效用货币*（Utility Currency），用于支付使用这台"世界计算机"的费用。

与脚本语言极为有限的比特币不同，以太坊被设计为一条通用可编程区块链，运行着一台能够执行任意复杂度代码的虚拟机。比特币的 Script 语言被刻意限制为对花费条件进行简单的真/假判断，而以太坊的语言则是*图灵完备*（Turing Complete）的——这意味着以太坊可以充当一台通用计算机。

2022年9月，以太坊通过 The Merge 升级进一步与比特币区分开来，将共识模型从工作量证明（PoW）转向权益证明（PoS）。这一重要变革不仅彰显了以太坊对降低环境影响的承诺——与其创新愿景相一致——还增强了其可扩展性和安全特性。

<a id="blockchain_components"></a>

#### 区块链的组件


开放公共区块链（通常）包含以下组件：

* 基于标准化"Gossip"协议的 P2P 网络，用于连接参与者、传播交易和包含已验证交易的区块
* 以交易形式表示的消息，代表状态转换
* 一套共识规则，规定何为有效交易、何为有效状态转换
* 根据共识规则处理交易的状态机
* 由密码学保护的区块链——作为所有已验证和已接受状态转换的日志
* 共识算法，通过迫使参与者协作执行共识规则来分散对区块链的控制权
* 博弈论上合理的激励机制（如 PoW 成本加区块奖励），在开放环境中为状态机提供经济安全保障
* 上述组件的一个或多个开源软件实现（"客户端"）

这些组件通常整合在单一软件客户端中。例如，比特币的参考实现由 Bitcoin Core 开源项目开发。以太坊最初也只需要单一客户端，但在转向 PoS 后，现在需要两个独立的客户端：一个负责共识，一个负责执行。以太坊没有参考实现，而是依赖参考规范——即 [Yellow Paper](https://ethereum.github.io/yellowpaper/paper.pdf) 中详述的数学描述，该规范在以太坊发展过程中持续更新。以太坊社区目前正在过渡到用 Python 编写的参考规范，分别用于[共识客户端](https://github.com/ethereum/consensus-specs)和[执行客户端](https://github.com/ethereum/execution-specs)。多个客户端团队根据参考规范独立构建。我们将在第三章深入探讨这一主题。

过去，我们用"区块链"一词作为上述所有组件的简称，代指涵盖所有这些特征的技术组合。然而如今，区块链种类繁多，特性各异。我们需要限定词来理解所讨论区块链的特征，例如*开放、公开、全球化、去中心化、中立、抗审查*等，以识别这些组件所赋予的"区块链"系统的重要涌现特性。

并非所有区块链都生而平等。尽管它们展现出大量共同属性，我们可以大致将区块链分为许可制与无许可制、公有与私有：

**无许可制（Permissionless）**
: 无许可制区块链（如比特币和以太坊）对任何人开放。这些去中心化网络允许任何人加入、参与共识过程、读写数据，通过透明性建立信任。

**许可制（Permissioned）**
: 许可制区块链限制访问，仅允许授权参与者加入网络并执行特定操作。

**公有（Public）**
: 公有区块链去中心化且对所有人开放，允许广泛参与网络活动，通过广泛分布和共识机制确保透明性。

**私有（Private）**
: 私有区块链将访问权限制于特定参与者群体，通常在组织内部或受信任的合作伙伴之间使用。

<a id="ethereum_birth"></a>

### 以太坊的诞生


所有伟大的创新都解决真实的问题，以太坊也不例外。当人们认识到比特币模型的威力，并试图将其应用扩展到加密货币之外时，以太坊的构想应运而生。但开发者面临一个两难抉择：要么在比特币之上构建，要么启动一条全新的区块链。在比特币上构建意味着必须在网络的设计约束内寻找变通方案。有限的交易类型、数据类型和数据存储大小限制了可以直接在比特币上运行的应用种类；其他一切都需要额外的链下层，而这立即抵消了使用公共区块链的诸多优势。对于需要更大自由度和灵活性且希望留在链上的项目，启动新区块链是唯一选择。但这意味着大量工作：引导所有基础设施组件、详尽的测试等等。

2013年底，年轻的程序员和比特币爱好者 Vitalik Buterin 开始思考如何进一步扩展比特币和 Mastercoin（一个在比特币之上提供基础智能合约功能的叠加协议）的能力。同年10月，Buterin 向 Mastercoin 团队提出了一个更通用的方案：用灵活的可脚本化合约（虽然不是图灵完备的）来替代 Mastercoin 的专用合约语言。尽管 Mastercoin 团队印象深刻，但这个提案过于激进，无法纳入他们的开发路线图。

2013年12月，Buterin 开始分享一份白皮书，勾勒出以太坊背后的理念：一条图灵完备的通用区块链。数十人看到了这份早期草案并提供反馈，帮助 Buterin 不断完善提案。

本书的两位原作者 Andreas M. Antonopoulos 和 Gavin Wood 博士都收到了白皮书初稿并提出了意见。Antonopoulos 对这个想法非常着迷，向 Buterin 询问了许多问题，包括使用独立区块链来强制执行智能合约的共识规则，以及图灵完备语言的影响等。Antonopoulos 持续关注着以太坊的进展，但当时他正处于撰写《Mastering Bitcoin》一书的初期阶段，直到很久以后才直接参与以太坊。而 Wood 是最早联系 Buterin 并贡献其 C++ 编程技能的人之一，后来成为以太坊的联合创始人、联合设计师和 CTO。

正如 Buterin 在其 ["以太坊史前史"](https://vitalik.eth.limo/general/2017/09/14/prehistory.html) 一文中所述：

> 当时的以太坊协议完全是我自己的创作。然而，从这里开始，新的参与者开始加入。迄今为止在协议方面最突出的是 Gavin Wood。

将以太坊的视角从"可编程货币"平台——基于区块链的合约可以持有数字资产并根据预设规则转移——转变为通用计算平台，这在很大程度上要归功于 Wood。这一变化始于侧重点和术语的细微调整，随着"Web3"理念日益受到重视，其影响变得更加深远。Web3 将以太坊视为一套去中心化技术的核心组件，另外两个是 Whisper 和 Swarm。从2013年12月开始，Buterin 和 Wood 不断完善和发展这个构想，共同构建了以太坊的协议层。

以太坊的创始人们构想的不是一条针对特定目的的区块链，而是一条通过*可编程性*来支持各种应用的通用区块链。核心理念是：通过使用以太坊这样的通用区块链，开发者可以专注于编写自己的应用程序，而无需实现 P2P 网络、区块链、共识算法等底层机制。以太坊抽象掉这些细节，为编写去中心化应用提供确定性和安全的环境。这种思维转变不仅让开发变得更容易，还从根本上拓展了区块链的能力边界。它为去中心化金融（DeFi）、NFT 和去中心化自治组织（DAO）等全新领域奠定了基础——这些在早期单一用途的区块链上是不可行的。

正如 Satoshi Nakamoto（比特币的匿名开发者）一样，Buterin 和 Wood 不仅发明了新技术，还以创新的方式将新发明与现有技术结合，并提供原型代码向世界证明他们的想法。

创始人们花了数年时间来构建和完善这一愿景。2015年7月30日，第一个以太坊区块被挖出，"世界计算机"开始服务于全世界。

<a id="development_stages"></a>

### 以太坊的发展阶段


以太坊的发展被规划为四个不同的阶段，每个阶段都会发生重大变化。每个阶段可能包含子版本，即*硬分叉*——以不向后兼容的方式改变功能。

四个主要发展阶段的代号分别是 Frontier、Homestead、Metropolis 和 Serenity。截至本文撰写时，我们正处于最后一个阶段：Serenity。Serenity 阶段又被细分为六个子阶段，代号分别为 The Merge、The Surge、The Scourge、The Verge、The Purge 和 The Splurge。

让我们深入了解这四个发展阶段及其主要目的：

**Frontier（2015年7月30日）**
: 在创世区块（第一个以太坊区块被挖出时）启动，Frontier 为矿工和开发者奠定了基础，使他们能够搭建挖矿设备、开始 ETH 代币交易，并在最小化的网络环境中测试去中心化应用（DApp）。最初，区块的 Gas 上限为五千，但在2015年9月被提升，允许进行交易并引入了"难度炸弹"。以太坊的*难度炸弹*是一种旨在随时间指数级增加挖矿难度的机制，最终使挖矿变得不可行，从而激励从原始的 PoW 共识过渡到目前使用的更节能的 PoS 模型。

**Homestead（2016年3月14日）**
: 在区块 1,150,000 启动，Homestead 通过关键协议更新（EIP-2、EIP-7 和 EIP-8）使以太坊更加安全和稳定。这些升级增强了开发者友好性，并为进一步的协议改进铺平了道路，尽管网络仍处于测试阶段。

**Metropolis（2017年10月16日）**
: 从区块 4,370,000 开始，Metropolis 旨在增加网络功能，促进 DApp 创建和整体网络效用。此阶段的重要分叉如 Byzantium、Constantinople 和 Istanbul 优化了 Gas 成本、增强了安全性，并引入了二层（L2）扩容解决方案。Byzantium 降低了挖矿奖励并实施了密码学条款，而 Constantinople 进一步优化了 Gas 成本并允许与未创建地址进行交互。Istanbul 使网络对分布式拒绝服务（DDoS）攻击更具韧性，并引入了零知识密码学证明（zk-SNARKs 和 STARKs）以提高可扩展性和隐私性。这些增强共同为以太坊 2.0 奠定了基础，代表了以太坊 1.0 的最后阶段。

**Serenity（2022年9月15日）**
: Serenity，通常被称为*以太坊 2.0*，是一次旨在将以太坊从 PoW 转变为 PoS 共识机制的重大升级。Serenity 专注于使以太坊更具可持续性，并能够处理不断增长的用户和应用。这一阶段解决了高能耗和网络拥堵等关键问题，为更强健、更高效的区块链铺平了道路。

Serenity 升级被划分为几个子阶段，每个子阶段针对网络演进的特定方面。虽然主要的四个发展阶段是按顺序实施的，但 Serenity 的子阶段是同时开发的。这种并行方式是对先前开发方法的战略性转变，之所以可行，是因为前四个阶段奠定的基础工作为 Serenity 子阶段的同步开发提供了必要条件。每个子阶段从不同方面改进以太坊链，彼此独立，从而实现更灵活和动态的升级过程：

**The Merge**
: The Merge 将以太坊主网与信标链（处理 PoS 共识的侧链）合并，正式将网络过渡到 PoS，显著降低能源消耗。

**The Surge**
: The Surge 引入分片，通过将网络分割成更小、可管理的部分来提高以太坊的可扩展性，从而实现更高的每秒交易量。

**The Scourge**
: The Scourge 解决中心化和抗审查问题，确保以太坊保持去中心化和开放的网络。

**The Verge**
: The Verge 实施 Verkle 树，减少节点所需的数据存储，从而提高网络效率和可扩展性。

**The Purge**
: The Purge 旨在减少以太坊上存储的历史数据，简化节点运营并降低网络拥堵。

**The Splurge**
: The Splurge 包含各种小型升级和优化，以确保在所有重大变更实施后以太坊能够平稳高效地运行。

> **译注**：原书写于2018年，以下为更新至2026年初的最新进展。

#### 最新升级进展

**Pectra（2025年5月7日）**
: 以太坊迄今功能最丰富的升级，包含11个 EIP。结合了 Prague 执行层硬分叉和 Electra 共识层升级。将每个区块的目标 blob 数量从3个增加到6个，最大值从6个提高到9个，有效地将 blob 吞吐量翻倍，进一步降低 L2 成本。

**Fusaka（2025年12月3日）**
: 引入 PeerDAS（点对点数据可用性采样），提高 Layer-2 rollup 的数据可用性，实现更快的扩容和更低的交易费用，同时保持较低的验证者要求。网络在2025年11月底将区块 Gas 上限提高到6000万。

**Glamsterdam（2026年上半年计划）**
: 专注于增强基础层执行和扩展 rollup 数据容量。主要目标包括并行处理、提高 Gas 上限和 ZK 证明验证，使以太坊更快、更便宜。

**Hegota（2026年下半年计划）**
: 针对状态膨胀问题。核心是实施 Verkle 树——一种新型数据结构，可减少节点的存储和处理需求。通过压缩证明数据，Verkle 树降低了硬件要求，使节点运营更加便捷。


<a id="general_purpose_blockchain"></a>

### 以太坊：通用区块链


最初的区块链——即比特币的区块链——追踪比特币单位的状态及其所有权。你可以将比特币视为一个分布式共识*状态机*，其中交易触发全局的*状态转换*，从而改变币的所有权。状态转换受共识规则约束，使所有参与者在挖出若干区块后（最终）就系统的共同（共识）状态达成一致。

以太坊也是一个分布式状态机。但与仅追踪货币所有权状态不同，以太坊追踪的是通用数据存储的状态转换——即可以存储任何能够表示为*键值元组*（Key-Value Tuple）的数据。键值数据存储通过某个键来引用任意值：例如，通过键 "Book Title" 引用值 "Mastering Ethereum"。从某种意义上说，这与大多数通用计算机使用的随机存取存储器（RAM）数据存储模型具有相同的用途。

以太坊有内存来存储代码和数据，并使用区块链来追踪这些内存随时间的变化。就像通用的存储程序计算机一样，以太坊可以将代码加载到其状态机中并*运行*，将结果状态变化存储在区块链中。与大多数通用计算机的两个关键区别在于：以太坊的状态变化受共识规则约束，且状态在全球分布。以太坊回答了这样一个问题："如果我们可以追踪任意状态，并对状态机编程，创建一台在共识机制下运行的全球计算机，会怎样？"

<a id="ethereum_components"></a>

### 以太坊的组件


在以太坊中，[区块链组件](#blockchain_components) 中描述的区块链系统组件更具体地包括：

**P2P 网络**
: 以太坊在*以太坊主网*上运行，可通过 TCP 端口 30303 访问，运行名为 [*ÐΞVp2p*](https://ethereum.org/en/developers/docs/networking-layer/) 的协议。

**共识规则**
: 以太坊最初的共识协议是 Ethash，一种在参考规范"黄皮书"中定义的 PoW 模型。随后在2022年9月的 The Merge 升级中演变为 PoS（见第15章）。

**交易**
: 以太坊交易是网络消息，包含（除其他内容外）发送者、接收者、值和数据负载。

**状态机**
: 以太坊的状态转换由*以太坊虚拟机*（EVM）处理——这是一个执行*字节码*（机器语言指令）的基于栈的虚拟机。被称为*智能合约*的 EVM 程序用高级语言（如 Solidity）编写，并编译为字节码以在 EVM 上执行。

**数据结构**
: 以太坊的状态作为*数据库*（通常使用 Google 的 LevelDB）本地存储在每个节点上，以名为 *Merkle-Patricia Trie* 的序列化哈希数据结构存储交易和系统状态。

**共识算法**
: 以太坊已从 PoW 过渡到 PoS 共识机制，以提高能源效率和可扩展性。在 PoS 中，验证者质押其加密货币以获得验证交易、创建新区块和维护网络安全的权利。以太坊的 PoS 是两种不同算法的融合：Casper the Friendly Finality Gadget (FFG) 和带有最新消息驱动 (LMD) 更新的 GHOST (Greedy Heaviest Observed Subtree)（详见第15章）。

**经济安全**
: 以太坊使用名为 Gasper 的 PoS 算法为区块链提供经济安全。我们将在第15章详细探讨 Gasper 的工作原理，包括其在最终性和验证者协调中的作用。

**客户端**
: 以太坊有多个可互操作的执行和共识客户端软件实现，其中最突出的是 *go-ethereum* (Geth) 和 Nethermind 用于执行层，Prysm 和 Lighthouse 用于共识层。

<a id="turing_completeness"></a>

### 以太坊与图灵完备性


当你开始了解以太坊时，会立即遇到*图灵完备*（Turing Complete）这个术语。人们说，与比特币不同，以太坊是图灵完备的。这究竟意味着什么？

这个术语源自英国数学家 Alan Turing，他被誉为计算机科学之父。1936年，他创建了一个计算机的数学模型：一个通过读写顺序存储器（类似于无限长的纸带）来操作符号的状态机。借助这个构造，Turing 为回答*通用可计算性*问题（是否所有问题都可解）提供了数学基础——答案是否定的。他证明了存在一类不可计算的问题。具体来说，他证明了*停机问题*（Halting Problem）——给定任意程序及其输入，判断该程序是否最终会停止运行——是无解的。

Turing 进一步定义：如果一个系统可以模拟任何图灵机，那么它就是*图灵完备*的。这样的系统被称为*通用图灵机*（Universal Turing Machine, UTM）。

以太坊在名为 EVM 的状态机中执行存储的程序，具备在内存中读写数据的能力，这使其成为一个图灵完备系统，因此是一台通用图灵机。在有限存储的约束下，以太坊可以计算任何图灵机能计算的算法。

以太坊的突破性创新在于：将存储程序计算机的通用计算架构与去中心化区块链相结合，从而创建了一台分布式的单状态（Singleton）世界计算机。以太坊程序在"各处"运行，却产生由共识规则保障的共同状态。

<a id="turing_completeness_feature"></a>

#### 图灵完备作为"特性"


听说以太坊是图灵完备的，你可能会认为这是图灵不完备系统所缺乏的*特性*。恰恰相反。图灵完备性非常容易实现；事实上，[已知最简单的图灵完备状态机](https://www.wolframscience.com/nks/p707--implications-for-mathematics-and-its-foundations/)只有4个状态、使用6个符号，状态定义仅22条指令长。确实，有时系统会被发现是"意外图灵完备"的（这里有一份[此类系统的有趣参考](http://beza1e1.tuxen.de/articles/accidentally_turing_complete.html)）。

然而，图灵完备性在开放访问系统（如公共区块链）中非常危险，正是因为前面提到的停机问题。例如，现代打印机是图灵完备的，可以接收导致其陷入死机状态的打印文件。以太坊是图灵完备的这一事实意味着任何复杂程度的程序都可以在以太坊上执行。但这种灵活性也带来了棘手的安全和资源管理问题。无响应的打印机可以关机再开机，但公共区块链不行。

<a id="turing_completeness_implications"></a>

#### 图灵完备性的影响


Turing 证明了你无法通过在计算机上模拟程序来预测程序是否会终止。简言之，我们无法在不运行程序的情况下预测其执行路径。图灵完备系统可以陷入*无限循环*——这是（过于简化地）描述永不终止程序的术语。创建一个永不结束的循环程序很容易。但由于初始条件和代码之间复杂的相互作用，无意的无限循环可能在毫无预警的情况下出现。在以太坊中，这带来了一个挑战：每个参与节点（客户端）必须验证每笔交易，运行其调用的所有智能合约。但正如 Turing 所证明的，以太坊无法在不实际运行（可能永远运行）的情况下预测一个智能合约是否会终止，或会运行多久。无论是有意还是无意，都可以创建一个智能合约，使其在节点尝试验证时永久运行。这实际上构成拒绝服务（DoS）攻击。当然，在毫秒级验证的程序和永远运行的程序之间，存在无限多令人头疼的情况：资源浪费、内存膨胀、CPU 过载的程序，这些都只是在白白消耗资源。在世界计算机上，滥用资源的程序就是在滥用全世界的资源。如果以太坊无法预先预测资源使用情况，它如何限制智能合约使用的资源？

为应对这一挑战，以太坊引入了名为 *Gas* 的计量机制。当 EVM 执行智能合约时，它会精确核算每条指令（计算、数据访问等）的成本。每条指令都有一个以 Gas 为单位的预定成本。当交易触发智能合约执行时，必须附带一定量的 Gas，用于设定该智能合约可消耗的计算上限。如果计算消耗的 Gas 超过交易中可用的 Gas，EVM 将终止执行。Gas 是以太坊实现图灵完备计算的同时限制程序资源消耗的机制。

下一个问题是：如何获取 Gas 来支付以太坊世界计算机上的计算？你在任何交易所都找不到 Gas。Gas 只能作为交易的一部分购买，且只能用以太币购买。以太币需要随交易一起发送，并且必须明确指定用于购买 Gas，同时附带一个可接受的 Gas 价格。就像在加油站一样，Gas 的价格不是固定的。Gas 为交易而购买，计算执行完毕后，任何未使用的 Gas 将退还给交易发送者。

<a id="DApp"></a>

### 从通用区块链到去中心化应用


以太坊最初被设计为一条可用于各种目的的通用可编程区块链。但很快，以太坊的愿景扩展为编程去中心化应用（DApp）的平台。DApp 代表比智能合约更宽广的视角。一个 DApp 至少包含一个智能合约和一个 Web 用户界面。更广义地说，DApp 是构建在开放、去中心化的 P2P 基础设施服务之上的 Web 应用程序。

DApp 至少由以下部分组成：

- 区块链上的智能合约
- Web 前端用户界面

此外，许多 DApp 还包含其他去中心化组件，例如：

- 去中心化（P2P）存储协议和平台
- 去中心化（P2P）消息传递协议和平台

从实践角度看，以太坊的 web3.js JavaScript 库将浏览器中运行的 JavaScript 应用与以太坊区块链连接起来。它最初包含一个名为 Swarm 的 P2P 存储网络和一个名为 Whisper 的 P2P 消息服务——这些工具使开发完全去中心化的 Web3 DApp 成为可能。尽管这种完全去中心化的 DApp 设计很有吸引力，但在随后的几年中并未获得太多关注。为了改善用户体验和促进用户采用，不得不接受妥协，如今与智能合约交互的中心化 Web2 网站已成为 DApp 的标准形式。

> **译注**：原书提到的 Whisper P2P 消息协议已被弃用，现已被 Waku 取代。Swarm 存储网络目前作为独立项目运行，不再是以太坊核心组件。现代 DApp 开发通常使用 ethers.js 或 viem 等更新的库。

<a id="evolving_WWW"></a>

### 互联网的第三个时代


2004年，*Web 2.0* 这个术语开始流行，用于描述互联网向用户生成内容、响应式界面和交互性的演进。Web 2.0 不是技术规范，而是描述 Web 应用新焦点的术语。

DApp 的概念旨在将互联网引向下一个自然演进阶段——将去中心化的 P2P 协议引入 Web 应用的各个层面。描述这一演进的术语是 *Web3*，意为 Web 的第三个"版本"。由 Gavin Wood 首先提出，Web3 代表了 Web 应用的新愿景和焦点：从中心化拥有和管理的应用转向基于去中心化协议的应用。

<a id="development_culture"></a>

### 以太坊的开发文化


到目前为止，我们讨论了以太坊在目标和技术上与其他区块链（如比特币）的区别。以太坊还有着截然不同的开发文化。

在比特币中，开发遵循保守原则：所有变更都经过仔细研究，以确保不会破坏现有系统。大多数情况下，只有向后兼容的变更才会被采纳。允许现有客户端"选择加入"，但如果他们决定不升级，也能继续运行。这种谨慎的方法与比特币的治理模式相一致——变更通过比特币改进提案（BIP）流程进行，这是一个刻意缓慢且共识驱动的管道，旨在保持稳定性。

相比之下，以太坊社区的开发文化聚焦于未来而非过去。其（并非完全严肃的）信条是"快速行动，打破常规"（Move fast and break things）。如果需要变更，即使意味着使之前的假设失效、破坏兼容性或强制客户端更新，也会执行。以太坊的治理体现了这种更具实践性的风格，协调工作通过频繁的 AllCoreDevs 电话会议公开进行，研究人员、客户端团队和生态系统利益相关者在会上讨论和协调即将到来的变更。这是一个更敏捷、更迭代的过程，以牺牲一些稳定性换取更快的创新节奏。

这对开发者意味着：你必须保持灵活，随时准备在某些基础假设发生变化时重建基础设施。以太坊开发者面临的一个重大挑战是将代码部署到不可变系统与开发平台仍在演进之间的内在矛盾。你无法简单地"升级"你的智能合约。你必须准备好部署新合约、迁移用户、应用和资金，然后重新开始。

讽刺的是，这也意味着构建更多自主权、更少中心化控制的系统这一目标尚未完全实现。自治和去中心化需要平台具备比你在未来几年可能从以太坊获得的更高的稳定性。为了"发展"平台，你必须准备好废弃并重启你的智能合约，这意味着你必须保留一定程度的控制权。

但积极的一面是，以太坊正在快速发展。几乎没有*琐碎争论*（Bikeshedding）的机会——这个词指的是因争论无关紧要的细节（比如如何在核电站后面建自行车棚）而耽误开发。如果你开始纠结于琐碎细节，可能会突然发现当你分心时，开发团队已经改变了计划，放弃自行车转而支持自动驾驶气垫船。

最终，以太坊平台的开发速度会放缓，接口会趋于稳定。但在此之前，创新是驱动原则。你最好跟上节奏，因为没有人会为你放慢脚步。

<a id="why_learn"></a>

### 为什么要学习以太坊？


区块链有着非常陡峭的学习曲线，因为它融合了多个学科：编程、信息安全、密码学、经济学、分布式系统、P2P 网络等等。以太坊大大降低了这一学习曲线的陡峭程度，让你可以快速上手。但在看似简单的表面之下，隐藏着更多内容。随着学习的深入，你总会发现新的复杂性层次和惊喜。

以太坊是学习区块链的绝佳平台，它以比任何其他区块链平台都快的速度构建起了庞大的开发者社区。与其他区块链相比，以太坊是*开发者的区块链*：由开发者为开发者打造。熟悉 JavaScript 应用的开发者可以进入以太坊并快速开始编写可运行的代码。在以太坊早期，经常能看到 T 恤衫上宣称只需五行代码就能创建一个代币。当然，这是把双刃剑——编写代码很容易，但编写*优质*且*安全*的代码却非常困难。

许多区块链项目，如 L2，都基于以太坊构建。学习以太坊有助于你更好地理解这些项目，并为你提供探索区块链世界进一步发展的工具。对于任何希望参与区块链技术最新进展的人来说，这些知识至关重要。

<a id="conclusion"></a>

### 结语


以太坊作为区块链领域的开创性平台脱颖而出。其图灵完备系统的设计允许创建具有复杂可编程逻辑的去中心化应用，超越了比特币较为简单的功能。

对于开发者和技术人员而言，理解以太坊为深入理解区块链技术及其潜在应用打开了大门。通过掌握以太坊，你将获得参与和贡献互联网持续演进的工具，使自己站在这个激动人心领域的最前沿。
